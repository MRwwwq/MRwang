# PRD §13 系统稳定性增强设计（正式定稿）

**核心能力：异步流水线架构 + 三级故障降级体系**

## 一、需求概述

通过全链路异步流水线解耦 + 分级故障自动降级机制，彻底消除主流程单点阻塞、服务雪崩、链路卡顿风险；保障盘中 30 分钟高频实时风控刷新稳定可靠，持续动态输出有效仓位系数，实现交易期零阻塞、故障不中断、异常不宕机的高可用风控架构。

## 二、异步流水线架构优化（彻底消除单点阻塞）

### 1. 全链路解耦异步化

整体风控链路采用消息驱动异步流水线：

| 服务 | 职责 | 模式 | Topic |
|:-----|:-----|:-----|:------|
| SIGNAL_EXTRACT | 信号提取 | 生产者 | `topic.signal.raw` |
| MISJUDGE_MATCH | 误判RAG匹配 | 消费+生产 | `topic.signal.matched` |
| RULE_SCORE_ENGINE | 四层打分核心 | 消费+生产 | `topic.risk.score` |
| POSITION_DECISION | 仓位决策输出 | 消费 | `topic.order.decision` |
| EVOLUTION_AGENT | 进化学习(旁路) | 异步旁路消费 | `topic.risk.score` |

所有非强依赖流程全部异步旁路执行，不阻塞实时打分与仓位输出主链路。

### 2. EVOLUTION_AGENT 完全异步旁路运行

记忆入库、样本复盘、参数迭代、日志归因全部旁路异步执行，不占用交易链路算力、不影响盘中实时刷新延迟、不造成任何主流程阻塞。

### 3. 流水线任务分级排队 + 错峰调度

| 优先级 | 任务类型 | 调度策略 |
|:------:|:---------|:---------|
| 高优 | 快照归档、FAISS入库 | 实时执行 |
| 普通 | 赛道复盘统计 | 盘中下行、盘后集中 |
| 低优 | 周度沙盒回测、报表 | 盘后错峰执行 |

盘中只做实时计算，盘后再做重算与进化迭代，彻底隔离算力压力。

### 4. 无阻塞实时风控输出能力

- 盘中 30 分钟周期全自动刷新，动态更新风险分值与仓位系数
- 双频机制：盘中高频信号(资金/舆情/价格) + 收盘全量重算(财务/周期/长期)
- 无论后台进化任务、向量任务、日志任务是否堆积，盘中交易决策永远实时、稳定、可用

## 三、三级故障降级高可用体系（兜底全场景故障）

| 等级 | 触发条件 | 处理策略 | 对交易影响 |
|:----:|:---------|:---------|:----------|
| **一级** | RAG/误判匹配异常 | 切静态加权模式，持续告警 | 不影响交易、不禁仓 |
| **二级** | 财务/基本面数据源中断 | 剔除失效因子，保留量价/资金/题材/周期 | 风控不中断 |
| **三级** | 大面积服务异常/链路瘫痪 | 动态模型停止，启用静态基线(单票≤12%/行业≤30%/总仓≤75%) | 禁止新开仓、只平不开 |

四级防护线：**自动探测 → 分级降级 → 日志归档 → 故障恢复自动切回**，形成完整的高可用闭环。

## 四、最终落地价值

1. 彻底消除单点阻塞、链路卡顿、服务雪崩风险
2. 盘中风控 100% 稳定可用，高频刷新不抖动
3. 故障场景自动降级、不崩、不停、不炸链路
4. 实时动态输出仓位系数，交易时段风控持续有效
5. 完全适配多智能体无人值守、全自动闭环运行架构

## 五、验收标准

| 验收项 | 验证方式 | 对应代码 |
|:-------|:---------|:---------|
| 1. 所有进化/记忆/复盘/日志任务不阻塞主打分 | EVO旁路handler + SHAP异步落盘 + 分级调度 | `handle_evolution_agent()` / `build_and_log_shap()` / `GradedTaskScheduler` |
| 2. 盘中30分钟刷新周期稳定，仓位系数动态更新 | 双频调度30min触发 + `run_intraday_refresh()` | `DualFrequencyScheduler` |
| 3. 任意单模块故障触发对应降级，主流程不中断 | `detect_fault_level` 三级判定链 | `FaultDegradationManager` |
| 4. 高并发/任务堆积场景下，实时风控无延迟无卡死 | `RateLimiter` 限流 + `BoundedSemaphore(5)` | `RateLimiter` |
